# 概觀

## 作業系統
1. 作業系統是一個隨時執行的程式。
2. 摩爾定律，每 18 個月電晶體數目增加一倍。

## 計算機組織
1. 靴帶式程式負責載入作業系統
2. 靴帶式程式保存於 ROM 中
3. 插斷是事件發生的通知
4. 插斷由硬體或軟體產生時
5. 插斷產生時，檢查中斷向量
6. 記憶體階層：CPU>暫存器>L1..3 Cache>L4 Cache>主記憶體>輔助記憶體（硬碟）
7. DMA 用於傳送資料到外部裝置

## 計算機架構
1. 多元處理系統的優點：增加產能、更經濟、更穩定
2. SMP 是最常見的多元處理系統架構
3. 分散式是指有多台 PC
4. 多元處理是指多顆 CPU
5. 多核心是指有多個 Core
6. 叢集式的定義並不明確

## 作業系統架構
1. 多元程式規劃要讓 CPU 隨時有事做
2. 分時（或多工）是多元程式規劃的實踐
3. 行程是執行中的程式
4. 工作排班，是指選擇程式載入記憶體
5. CPU排班，是指選擇程式執行
6. 置換，是指行程，在記憶體與硬碟中移動的過程
7. 讀取與儲存，是指程式（靜態檔案），在記憶體與硬碟中移動的過程
8. 執行，是把程式（靜態檔案）變成行程的過程
9. 虛擬記憶體，使行程不必全部都放在實體記憶體中

## 作業系統操作
1. 陷阱是軟體錯誤插斷
2. 作業系統有雙模運作，分別是核心模式、使用者模式
3. 計時器，進入使用者模式時，避免當機用，會定期切回核心模式

## 作業系統管理的項目
1. 程式不是一個行程
2. 作業系統要管理行程
3. 作業系統要管理記憶體
4. 作業系統要管理檔案系統
5. 作業系統要管理 I/O

# 作業系統結構

## 作業系統介面
1. 使用者介面，包含 3 種：CLI 命令列介面（文字介面）、批次介面、GUI 圖形介面
2. 行程間通信，包含 2 種：共享記憶體、訊息傳遞

## 使用者介面
1. 命令直譯程式（又稱 shell）
2. 圖形介面

## 雙模運作
1. 行程可以執行系統呼叫
2. 系統呼叫使用核心模式

## 系統呼叫
1. 系統呼叫，包含 6 種：行程控制、檔案管理、裝置管理、資訊維護（取得資料）、通信（行程通信）、保護（權限管理）

## 2.7 作業系統設計
1. 作業系統結構，包含 5 種：簡單結構、分層方法、微核心、模組、混合系統
2. 簡單結構，優點：簡單；缺點：容易崩潰
3. 分層方法，優點：簡單、易除錯；缺點：難定義、沒效率
4. 微核心，優點：容易擴展；缺點：功能增加容易低效
5. 模組，中庸之道
6. 混合系統，截長補短

# 行程管理（行程）

## 行程概念
1. 行程是 CPU 執行的運作項目
2. 行程，包含 4 個部分：程式碼、資料、推積、堆疊
3. 行程的狀態，包含 5 個部分：新產生、執行、等待、就緒、結束
4. 行程狀態圖
5. 每個行程都有 PCB 用來儲存行程資訊
6. 每個行程可以有一個或多個執行緒

## 行程排班
1. 行程排班，包含 3 種：長程排班（工作排班）、中程排班（置換）、短程排班（CPU排班）
2. 長程排班經常沒有或很少（UNIX 及 Windows 都沒有）
3. 內容轉換，是指CPU改變行程時，保存舊行程資訊、讀取新行程資訊的動作

## 行程間的關係
1. PID 是用來識別行程的
2. 行程可以產生子行程，產生後則稱為父行程
3. 父行程中止，子行程也會中止
4. POSIX 函數 fork() 可以產生子行程，其中子行程得到 0；而父行程得到子行程 PID
5. POSIX 函數 exec() 可以載入二進制檔案覆蓋程式，而且自動執行
6. POSIX 函數 wait() 可以將行程自己移出就緒佇列，變成等待狀態（等待子行程完成，父行程可以得到子行程 PID）
7. POSIX 函數 exit() 可以結束行程自己
8. 殭屍行程，是指子行程已經完畢，但保留 PCB 等父行程呼叫 wait() 函數的狀態
9. 孤兒行程，是指父行程沒有呼叫 wait() 而結束，這時子行程就會直接被根行程維護
10. Windows 的 API 長得很醜

## 行程間合作
1. 獨立行程，是不共用資料的行程
2. 行程合作的優點有 4 項：資訊共享、加速運算、模組化、方便性
3. IPC 包含 2 種模式：共享記憶體、訊息傳遞
4. 生產者與消費者範例
5. 訊息傳遞，分成 2 種：直接聯繫、間接聯繫
6. 訊息傳遞，又可分成 2 種：等待、非等待

## 行程跨網路合作
1. Client-Server 通信，分成 3 種：Socket、RPC、Pipe
2. Socket 是 Client 與 Server 互相傳送資料
3. RPC 是 Client 去執行 Server 的程式並得到回傳資料
4. Pipe 相當於行程共同讀寫 1 份類似檔案的資料
5. Pipe 分成 2 種：普通管道（Windows 又稱匿名管道）、命名管道（UNIX 又稱為 FIFO）
6. 普通管道只能單向、父-子行程間，同一台機器
7. UNIX 的命名管道，允許雙向傳送（半雙工）、非父-子行程、不可跨機器、位元資料
8. Windows 的命名管道，允許雙向傳送（全雙工）、非父-子行程、可跨機器、位元資料與本文資料

# 行程管理（執行緒）

## 執行緒概念
1. 執行緒是 CPU 執行的基本單位
2. 執行緒獨立的部分包含 ID、PC、暫存器、堆疊
3. 執行緒共用的部分程式碼、資料（全域變數）、作業系統資源

## 平行運算加速
1. 阿姆達爾定律

## 執行緒對應
1. 執行緒分成 2 種：使用者執行緒、核心執行緒
2. 執行緒對應的模式有 3 種：多對一模式、一對一模式、一對多模式
3. Linux 與 Windows 都是一對一模式

## 執行緒實作
1. 多執行 API 有 3 種：POSIX Pthreads、Windows、Java
2. 執行緒池

## 信號
1. UNIX 用信號來通知行程，事件的發生
2. 系統預設的信號處理器與使用者的信號處理器
3. 信號可以送給行程或執行緒

# 行程管理（排班）

## 行程排班
1. 這是短程排班
2. 為了實現多元程式規劃
3. 同一時間單個 CPU Core 只能執行同一工作
4. 排班佇列通常儲存 PCB
5. CPU 的排班決策發生在：行程從執行變等待（I/O 發生）、行程從執行變就緒（插斷發生）、行程從等待變就緒（I/O 結束）、行程中止
6. 不可搶先排班，行程必須自己離開執行狀態，即 I/O 發生、行程中止
7. 針對 I/O 結束的情況，例如 A 行程在執行中、就緒佇列為空；此時 B 行程 I/O 結束變成就緒狀態，如果 B 可以搶先 A 就是可搶先。

## 評估排班
1. CPU 使用率：讓 CPU 盡可能忙碌
2. 產量：單位時間完成的行程，越大越好
3. 回復時間：行程多少時間可以完成，越小越好
4. 等候時間：行程要等多久才可以被執行，越小越好
5. 反應時間：讓使用者感覺有反應的時間，越小越好
6. 學者說交談式系統，應使反應時間最小化
7. 我們目前考慮的是平均回復時間、等候時間

## 行程排班算法
1. FCFS 先來先做排班演算法（護送現象、不可搶先）
2. SJF 最短工作先做排班演算法（估測時間、可或不可搶先）
3. 優先權排班演算法（飢餓、老化、可或不可搶先）
4. RR 循環排班演算法（效能取決時間量、可搶先）
5. 多層佇列排班演算法（每個佇列都有自己的排班法則、不允許行程改變層次）
6. 多層回饋佇列排班演算法（類似多層佇列排班演算法、但允許行程改變層次）

## 多執行緒排班
1. PCS 行程競爭範圍，是指使用者執行緒搶核心執行緒
2. SCS 系統競爭範圍，是指核心執行緒搶 CPU 執行

## 多處理器排班
1. 處理器親和性，是指行程執行在一個 CPU 上後，不會跑到其他 CPU 上
2. 處理器親和性，分成 2 種：軟性親和性、硬性親和性
3. 軟性親和性為盡力安排而非強制，硬性親和性為 CPU 不對則無法執行
4. 負載平衡，工作平均分配到多個 CPU
5. 負載平衡，分為 2 種：推轉移、拉轉移
6. 推轉移，是指檢查當前載入，不平衡則搬移 CPU 執行的行程（主動、週期性）
7. 拉轉移，是指閒置 CPU 去拉忙碌 CPU 的行程，這個行程在等候中（被動）
8. 處理器親和與負載平衡剛好背道而馳
9. 記憶體停滯，指 CPU 讀取記憶體，結果資料不可用（快取失誤之類）
10. 多執行緒處理分成：粗糙、精緻
11. 粗糙的多執行緒處理，執行緒切換發生在記憶體停滯
12. 精緻的多執行緒處理，執行緒切換在每個指令週期邊緣

## 即時系統排班
1. 即時系統，分成 2 種：軟即時系統、硬即時系統
2. 軟即時系統，很急會先做，但不保證期限內完成
3. 硬即時系統，很急會先做，且保證時間內完成
4. 潛伏期，分成 3 種：事件潛伏期、中斷潛伏期、分派潛伏期
5. 事件潛伏期，是指事件發生到它被服務的時間
6. 中斷潛伏期，是指發生中斷到開始執行 ISR 的時間
7. 分派潛伏期，是指停止一個行程到啟動另一個行程的時間
8. 單調速率排班演算法（週期越短優先權越高、靜態優先權最佳方法）
9. EDF 最早截止期限優先排班演算法（截止期限越早優先權越高、動態優先權最佳方法）
10. 計算CPU使用率，可以確定有沒有解

# 行程管理（同步）

## 同步問題
1. 生產者-消費者模型
2. 競爭情況，兩個行程同時處理變數導致數值錯誤

## 臨界區間
1. 臨界區間，是指多個行程共同使用資源的區間
2. 臨界區間問題，是指設計一套協定使行程可以合作
3. 臨界區間問題，有 3 項要求：互斥、進行（不死鎖）、有限等待（不餓死）

## 解決方案：Peterson
1. Peterson 解決方案，想進入臨界區段時，將優先權交給對方

## 解決方案：硬體支援
1. 不可中斷的單元指令

## 解決方案：互斥鎖
1. 互斥鎖，包含 2 個函數：獲得 acquire、釋放 release
2. 獲得與釋放函數必須是不可中斷的單元指令
3. 自旋鎖，是指某行程鎖住時，其他行程盤旋於忙碌等待狀態的互斥鎖

## 解決方案：號誌
1. 號誌，包含 2 種：計數號誌、二元號誌
2. 號誌是互斥鎖的強化版
3. 號誌，實作 2 個函數：等待 wait、信號 signal
4. 號誌可以被用來處理非同步先後順序執行
5. 等待函數 wait 執行時，將行程閉鎖，改變成等待狀態
6. 信號函數 signal 執行時，將閉鎖行程喚醒
7. 號誌可能誘發死結
8. 優先權倒置問題
9. 優先權繼承協定可以解決優先權倒置

## 經典同步問題
1. 有限緩衝區問題
2. 讀取-寫入者問題
3. 哲學家進餐問題

## 監督程式
1. 監督程式，是指同步解決方案 ADT 化
2. ADT 把問題的解決方案封裝
3. ADT 是防呆用，程式設計師常常寫錯程式

# 行程管理（死結）

## 死結概念
1. 死結，是指大家都佔有部份資源，且在等候其他資源，但彼此都等不到
2. 例證，是指有多少資源可以使用（不是多少種）

## 死結特性

1. 死結中的行程不會完成執行
2. 死結形成的條件，包含 4 個：互斥、佔用與等候、不可搶先、循環式等候
3. 互斥，是指至少存在一種資源是不可共用的
4. 佔用與等候，是指至少存在一個行程已經佔用一個資源，且正在等候其他資源
5. 不可搶先，是指資源不能被搶先
6. 循環式等候，行程彼此等待，形成一個環狀等候的狀態
7. 資源配置圖，包含 3 個部件：行程節點、資源節點、要求邊、分配邊
8. 死結產生時，資源配置圖必有循環（條件四），但不是有循環就有死結（要看其他條件）
9. 資源配置圖有循環且資源只有單一例證時，死結產生

## 死結處理
1. 死結處理，方法有 3 種：預防或避免死結、恢復死結、無視

## 預防死結
1. 預防死結的方法之一，排除佔用與等候，執行前要取得所有需要的資源（資源使用率低、可能產生飢餓）
2. 預防死結的方法之一，排除佔用與等候，未佔有資源才能要求資源（資源使用率低、可能產生飢餓）
3. 預防死結的方法之一，排除不可搶先，使資源可以被搶先
4. 預防死結的方法之一，排除循環式等候，遞增要求，這時循環點會斷開

## 避免死結
1. 行程額外提供資源最大需求的數量
2. 安全狀態是指，系統能分配資源給各行程，而仍避免死結的狀態
3. 資源配置圖演算法，檢查圖中的循環，有循環則不配置（且不適用多例證資源）
4. 銀行家演算法，包含 4 個資料結構：可用資源向量、最大需求矩陣、佔用矩陣、需求矩陣（最大需求矩陣 - 佔用矩陣）
5. 銀行家演算法，包含 2 個算法：安全演算法（確認）、資源要求演算法（配置）
6. 安全演算法，包含 2 個資料結構：資源暫存向量、完成行程向量
7. 安全演算法，包含 4 步：初始化、模擬配置、釋放配置、檢查結果
8. 安全演算法，複雜度 O(nmn) 因為找每個行程（n）配置資源（m）最後檢查每個行程是否完成（n）
9. 資源要求演算法，包含 1 個資料結構：傳入的要求向量
10. 資源要求演算法，包含 4 個步：檢查最大需求、調整資料、安全檢查、狀態回溯
11. 資源要求演算法，複雜度為 O(3m + nmn + 3mn) 包含檢查最大資源及調整資源（2m）且檢查安全（nmn）然後回溯（m + 3mn）

## 偵測死結
1. 檢查死結是否發生
2. 單一例證，檢查資源配置圖中的環即可
3. 多個例證，靠偵測演算法（類似安全演算法）
4. 偵測演算法，全局包含 3 個資料結構：可用資源向量、佔用矩陣、要求矩陣（注意不是需求矩陣）
5. 偵測演算法，局部包含 2 個資料結構：資源暫存向量、完成行程向量
6. 偵測演算法，包含 3 個步驟：初始化、模擬完成要求、模擬釋放配置、檢查結果
7. 如果偵測演算法無法模擬完成要求則陷入死結，沒有完成的行程全在死結狀態

## 恢復死結
1. 恢復死結，有 2 種方法：取消死結中的所有行程、一次取消一個直到死結消失
2. 取消死結中的所有行程，代價很大，因為有些行程可能計算很久了
3. 一次取消一個直到死結消失，需要很多時間，因為偵測死結演算法耗時龐大
4. 恢復死結也可以透過資源撤回的方法

# 記憶體管理

## 記憶體概念
1. 記憶體具有階層性
2. 每一個行程有個別的記憶體空間
3. 每一個行程的合法存取，倚靠 2 個暫存器：基底暫存器、界限暫存器
4. 記憶體位置與行程連結，可在 3 種時間中完成：編譯時間、載入時間、執行時間
5. 大部分一般用途的作業系統，都使用執行時間確認記憶體的方法
6. 邏輯位置，是指 CPU 產生的位置（也稱虛擬位置）
7. 實體位置，是指記憶體單元所看到的位置
8. MMU 負責虛擬位置與實體位置的對應
9. 重定位暫存器，就是基於 MMU 中基底暫存器的別名
10. 使用者程式永遠無法看到實體上的位置
11. 靜態連結，在開發階段，會把使用到的外部資源都加入執行檔中
12. 動態連結，在執行階段，會根據記號載入程式到記憶體中執行（已經知道型別，可以由編譯器檢查）
13. 動態載入，在執行階段，會載入程式進記憶體執行（比較像是載入執行檔）

## 置換
1. 置換，是指行程被移到備份儲存體然後在移回記憶體的過程
2. 置換時間的計算，如果行程佔 100MB 備份儲存體傳輸速率 50MB 置換時間為 2 秒，因為同時換入跟換出總共 4 秒
3. 置換的內容轉換很慢，大多數時間都在傳輸
4. 置換的行程必須是閒置狀態，也可以不用全部置換
5. 標準置換沒有用在現代的作業系統中（因為記憶體很大，所以當記憶體不足會被啟動）
6. 行動裝置沒有置換

## 管理策略與問題
1. 連續記憶體配置，是早期配置記憶體的策略
2. 界限暫存器保存邏輯位置、重定位暫存器保存實體位置
3. 存取時，先判斷界限暫存器，再處理重定位暫存器（如果顛倒，非法存取會多做一次加法）
4. 分割是一種原始的記憶體配置（目前已不再使用）
5. 分割的配給策略，包含三種：最先配合、最佳配合、最差配合
6. 通常最先配合效能比最佳配合快一些
7. 斷裂又分成 2 種：外部斷裂、內部斷裂
8. 外部斷裂，是指記憶體經過載入移出，出現一些空間太小，以致於不太可用的小區間
9. 內部斷裂，是指配置的一些固定空間中，存在沒有使用完的空間
10. 聚集，是解決外部斷裂的方法之一
11. 允許行程記憶體不連續，是解決外部斷裂的方法之二
12. 允許行程記憶體不連續，有 2 種方法：分段、分頁

## 分段方法
1. 分段允許一個行程的記憶體不連續
2. 分段無法解決外部斷裂與聚集的需要（因為分段的大小不是固定的）
3. 分段透過（分段號碼、偏移量）存取某個記憶體
4. 分段方法的硬體，保存分段表，用以查詢分段界限、分段基底
5. 分段號碼是分段表的註標，然後檢查偏移量存取記憶體

## 分頁方法
1. 分頁允許一個行程的記憶體不連續
2. 分頁可以解決外部斷裂與聚集的需要（因為分頁的大小是固定的）
3. 分頁方法中，實體記憶體打散稱為欄（或框架）、虛擬記憶體打散稱為頁
4. 分頁方法透過（頁數、頁偏移量）存取某個記憶體
5. 分頁方法保存一張分頁表，記錄著基底位置
6. 頁數是分頁表的註標，然後檢查頁偏移量存取記憶體
7. 頁的大小通常都是二的次方
8. 分頁方法會導致內部斷裂
9. 每一個行程都有自己的分頁表、分欄表（空白欄表）
10. 原始分頁方法，因為分頁表在記憶體中，所以每次存取要讀兩次記憶體，導致效能低落
11. TLB 轉譯旁邊緩衝區負責解決紛頁低落的問題
12. TLB 失誤，就要到記憶體存取分頁表
13. TLB 有效記憶體存取時間為「命中率 * 一次記憶體存取時間 + 失誤率 * 兩次記憶體存取時間」
14. CPU Cache 跟 TLB 負責的部分不同，前者針對資料、後者針對頁存取
15. 分頁表通常會加上有效-無效位元
16. 分頁方法可以共用程式碼

## 分頁表結構
1. 近代實體記憶體非常大，所以為了減少查詢分頁表的時間，需要分頁的結構化技巧
2. 分頁表結構化的技巧，常見有 3 種：階層式分頁表、雜湊分頁表、反轉分頁表
3. 階層式分頁表，透過層層查詢處理大空間
4. 雜湊分頁表，利用雜湊函數找分頁
5. 反轉分頁表，原本分頁表的註標是虛擬位置，反轉成註標為實體位置
6. 反轉分頁表由系統維護，只有一份；其他結構的分頁表都由行程維護，有許多份

# 虛擬記憶體

## 記憶體概念
1. 虛擬記憶體，是一種允許行程不必完全存在於記憶體中而被執行的技術
2. 邏輯位置對應著虛擬記憶體位置（即原本的邏輯記憶體）

## 需求分頁
1. 需求分頁，是指不用整個分頁都載入，只載入需要的分頁
2. 虛擬記憶體普遍採用需求分頁
3. 行程置換與分頁置換是不同的程式負責的（但行程也被抽象地存在頁中，所以行程置換同時會發生分頁置換）
4. 分頁錯誤，是指某一頁被存取，但不在實體記憶體中
5. 純粹需求分頁，是指直到需要某頁，才將其載入置記憶體中
6. 有效存取時間，是「(1 - 分頁錯誤率 ) * 記憶體存取時間 + 分頁錯誤率 * 分頁錯誤的時間」

## 寫入複製
1. 寫入複製，是父-子行程共用分頁，直到行程寫入資料到分頁中時複製備份
2. 寫入複製是很常見的技巧

## 分頁替換演算法
1. 過度配置，是指把實體記憶體塞得滿滿的，以至於突然需要空白欄但沒有的情況（多元程式規劃）
2. 過度配置的處理方法，有 2 種：分頁替換、行程置換
3. 分頁替換時，通常需要二倍記憶體存取時間
4. 分頁替換時，使用修改位元（髒位元）使沒被修改的分頁可以直接蓋掉，而不需要存回輔助記憶體，節省一半的時間
5. 分頁替換是需求分頁法的基礎
6. 需求分頁，要 2 個演算法：欄的配置演算法、分頁替換演算法
7. 參考串，是一種表示法，評估分頁替換演算法時使用
8. 分頁替換演算法，介紹有 3 種：FIFO、OPT、LRU
9. FIFO 在替換時，總是選擇最老的一頁換出
10. FIFO 會遭遇畢雷地異常（提供配置的欄數增加，結果分頁錯誤率也增加）
11. OPT 是最佳分頁替換演算法，且不會遭遇畢雷地異常
12. OPT 在替換時，總是選擇未來最長時間不會用到的頁
13. OPT 實作很難（類似行程的 SJF）需要預知未來
14. LRU 是近似 OPT 的演算法
15. LRU 在替換時，總是選擇近來最少使用的頁
16. LRU 可以透過 2 種方式實作：計數器、堆疊
17. 近似 LRU 的演算法共有 3 種：額外參考位元演算法、第二次機會演算法、加強第二次機會演算法
18. 額外參考位元演算法，設定整數最高位，為最近一次的使用標誌，每次右移一位元，選擇最小值的頁替換掉
19. 第二次機會演算法，是一種 FIFO 算法，選擇後如果標誌為 0 則替換，但標誌為 1 時再給一次機會，並將標誌歸零
20. 加強第二次機會演算法，像第二次機會算法，但新增修改標誌，這個算法傾向選擇沒有修改的頁（節省寫回輔助記憶體的時間）
21. 計數基礎分頁替換算法，有 2 種：LFU、MFU
22. LFU 在替換時，選擇最不常使用的頁
23. MFU 在替換時，選擇最常使用的頁
24. 這節總共介紹 8 種頁替換算法：FIFO、OPT、LRU、額外位元、二次機會、加強二次機會、LFU、MFU

## 欄配置演算法
1. 作業系統本身也需要一些欄
2. 欄的配置演算法，包含 2 種：同等分配、比例分配（按行程大小）
3. 分頁替換法可以分成 2 大類：全體替換法、區域替換法

## 輾轉現象
1. 輾轉現象，是指非常頻繁的分頁錯誤及多元程式規劃的失誤，導致CPU使用率極低
2. 輾轉現象過程是，行程的欄太少，等待頁替換時，就緒佇列空了，CPU排班引入行程提升多元程式規劃程度，結果惡性循環（欄是有限的）
3. 防止輾轉現象的發生，有 2 種方法：工作組策略、PFF
4. 工作組策略可以計算一個行程短期內所需的欄，讓配置的數量達到最低標準
5. PFF 在每個行程都增加錯誤頻率的數值，超過上限就多配一個欄、低過下限就取走一個欄

## 其他因素
1. 預先分頁，是指一開始載入行程就同時載入一些分頁
2. 程式設計師不太會去設定分頁大小
3. 計算程式迴圈的分頁錯誤次數

# 大量儲存結構

##  概觀
1. 硬碟是最常見的輔助記憶體
2. 硬碟的區域大小分成 3 種：磁柱>磁軌>磁區
3. 硬碟速度單位是「每分鐘」多少轉（RPM）
4. 硬碟的傳輸速率是硬碟速度的瓶頸

##  硬碟排班
1. 硬碟排班算法，大致有 6 種：FCFS、SSTF、SCAN、C-SCAN、LOOK、C-LOOK
2. FCFS 先來先做演算法
3. SSTF 最短尋找時間先做演算法
4. SCAN 自一端到另一端掃描（又名升降梯演算法）
5. C-SCAN 是 SCAN 的變種，只走單向
6. LOOK 是 SCAN 的變種，它不走到底
7. C-LOOK 是 C-SCAN 的變種，它不走到底
