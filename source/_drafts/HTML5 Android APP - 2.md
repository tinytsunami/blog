# 開發軟體
開發軟體不是這裡的主要重點，
因為軟體開發的需要技術範圍太廣很難一次到位。

這裡使用筆者隨便寫的遊戲專案當作範例，
下面是軟體開發的簡易介紹跟流程：

* 系統規劃：大致規劃需要完成的功能。
* 選擇套件：如果需要引入別人寫好的套件，則需要好好選擇。
* 設計實作：設計實作上的細節跟規範，方便之後的維護。
* 撰寫程式：實際開發軟體。

{% note warning %}
這裡筆者的專案不太大，所以看起來很簡單；其實大型軟體專案的開發步驟要複雜得多。
{% endnote %}

## 系統規劃
對於小型專案來說，甚至直接跳過這個階段都可以，
不過隨著專案的變大，系統規劃的重要性就越變越重要了。

筆者以一個極小型的射擊遊戲為例，
大致上先想好有哪些場景，每個場景的重點在哪裡，具體請參考圖 1。

![圖 1、系統規劃示意圖](https://i.imgur.com/bbBaJZQl.png)

{% note info %}
開發的過程中，有可能會來改動原先的設計，這是正常現象。
{% endnote %}

## 選擇套件
這個過程並不是必須的，要看專案是否使用套件為準，
但其實這個時代不用任何套件、API或函式庫之類的東西，
實作應用程式似乎已經不太可能了。

筆者遊戲開發以 PIXI.js 作為 HTML5 Canvas 的繪圖套件，
選擇有很多需要考慮的地方：

* 套件的授權（license）是什麼？
* 套件的文件（documention）與教程（tutorials）是否完整？
* 套件的使用社群是否活躍？
* 你「真的」需要這個套件嗎？
* ……

PIXI.js 是一款基於 MIT 授權的繪圖套件，而 MIT 授權的條款是很寬鬆的；
此外 PIXI.js 有活躍的社群、完整的文件及教程。

而且筆者玩的 RPG Maker MV 就是基於 PIXI.js 建立的，
也不算是太陌生的繪圖套件了。

{% note info %}
保持簡單是一個很重要的原則，採用之前想清楚以避免損失。
{% endnote %}
