# BUG 回報方法

基本上所有插件正式發布前，我會針對所有功能做一次複查，但有時插件指令太多（或是參數定義域太廣）沒辦法檢查很全面，如果有發現任何 BUG 可以回報給我，雖然畢業後有點忙，不過筆者還是會排時間修復。

為了加速檢查過程，請參考 BUG 回報流程：

1. 請到 [GitHub Issue](https://github.com/tinytsunami/RM-Plugins/issues) 頁面。

2. 點擊「New Issue」並提供以下資訊：

{% note info %}
可以複製貼上下列文字，並取代 `...` 的部分，但請不要複製行號；
同時請參考 [範例回報](https://github.com/tinytsunami/RM-Plugins/issues/1) 頁面。
{% endnote %}

{% codeblock %}
### 插件名稱（提供可識別的插件名稱）
...

### 錯誤訊息（請直接截圖遊戲畫面）
...

### 控制台訊息（請按下 F12 點選 Console 頁籤截圖）
...

### 相關操作說明（請描述如何觸發 BUG 的）
...

### 其他內容（如有請填寫，比方說要趕上架很急，我會優先處理）
...
{% endcodeblock %}

螢幕截圖的部分，使用 Windows 10 的使用者，可以透過 `Shift + Windows + S` 來截圖，截圖後，可以直接在 GitHub Issue 內使用「貼上」功能，就會得到類似 `![image](...)` 的文字（該字串發布後將會轉換成圖片）。

# 關於授權

所有插件都透過「Timi」名義發布，若使用本插件，相當於您同意下列授權：

```
# 由 Timi 授權條款發布
  - 基於 MIT 授權
  - 且您同意您的遊戲或作品不能使用非法素材
    （偷竊、未授權使用、侵害授權條款或其他侵害素材作者權益的行為）
```

授權釋義：

* 若您的遊戲或作品 **沒有** 使用任何非法素材，則本授權相當於 [MIT 授權](https://zh.wikipedia.org/zh-hant/MIT%E8%A8%B1%E5%8F%AF%E8%AD%89)
* 若您的遊戲或作品 **有** 使用任何非法素材，則 **請勿使用本插件**
* 使用本插件 **不必要** 向筆者通知或將筆者放入結束字幕（Closing Credit）中，但如果有，我會很高興
* 授權的更新不溯及既往。若未來條款更新，舊版插件仍使用舊版授權條款
